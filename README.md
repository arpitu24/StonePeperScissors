Developed a responsive browser-game version of the classic “Stone/Paper/Scissors” challenge, enabling a user to play against a computer opponent. Built the project from scratch using vanilla JavaScript for game logic, HTML5 for structure, and CSS3 for styling and responsive layout. This project demonstrates my ability to apply core web development concepts—DOM manipulation, event handling, conditional logic, state management, and responsive design—to create a fun, interactive application.

Key Features & Highlights:

Developed intuitive UI: allows the user to select “Stone”, “Paper” or “Scissors”, triggers the computer’s choice, and immediately displays the outcome (win/lose/draw).

Implemented randomized computer move logic, and clear outcome evaluation (e.g., stone beats scissors, paper beats stone, etc.).

Maintained score tracking across rounds, visually updated the scoreboard after each move.

Ensured responsive behaviour across device sizes (desktop + mobile) by using CSS flex/grid and media queries.

Structured code for readability and future extensibility: separated concerns (UI updates vs game logic), used meaningful variable/function names, commented key logic sections.

Strengthened debugging and testing skills: manually tested all possible move-combinations, handled edge cases (e.g., same move → draw).

Version-controlled the codebase using Git, and published publicly in a GitHub repository to showcase my work (good for portfolio visibility).

Why this project matters:
As someone starting my developer journey with JavaScript, this project provided a concrete exercise in end-to-end front-end development: from HTML/CSS layout building, through event-driven JavaScript logic, to producing a user-facing interactive experience. It showcases my ability to deliver a complete, functioning application with attention to UI/UX and code structure. This will help employers see that I can translate learning into a tangible product and that I have foundational web development capabilities.

What I learned / Skills gained:

JavaScript fundamentals: variables, functions, conditionals, loops, event listeners, randomization.

DOM manipulation: selecting elements, updating inner text/style, responding to user clicks.

UI design & responsive CSS: layout structuring, mobile-first considerations, visual feedback for user interaction.

Game-logic design: mapping state transitions, handling user vs. computer decisions, updating scores.

Version control (Git/GitHub): repository creation, commits, documentation (README).

Problem-solving: breaking down a real-life idea into discrete tasks (UI layout → game logic → result display → score update → responsive tweaks).
